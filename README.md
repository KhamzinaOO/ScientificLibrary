## Разработка информационной системы интеллектуального поиска в научных электронных библиотечных ресурсах

Информационная система интеллектуального поиска научных публикаций, разработанная в рамках научно-исследовательской работы и представленная для оценки 19.06.2025.

Цель проекта заключалась в исследовании проблематики релевантного поиска в электронных научных библиотеках и представлении математической модели для современной оценки значимости научной публикации.

Проект объединяет клиентское приложение на Kotlin Multiplatform, серверную часть на Spring Boot и общий модуль моделей. Система позволяет искать научные публикации, уточнять выдачу фильтрами, просматривать подробную информацию и изменять параметры ранжирования.

**Возможности:**
- поиск публикаций по текстовому запросу;
- расширенная фильтрация по авторам, журналам, площадкам, типам публикаций, году, датам, цитированиям, h-index и Open Access;
- автодополнение значений фильтров;
- ранжирование публикаций с учетом BM25, цитируемости, h-index авторов, влиятельных цитирований и возраста публикации;
- просмотр подробной карточки публикации;
- переход к открытому PDF при наличии ссылки;
- настройка весов модели ранжирования через клиентское приложение.

Клиентская часть построена на Compose Multiplatform и MVI-подходе: пользовательские события обрабатываются во ViewModel, состояние экранов хранится в StateFlow, данные поступают через репозитории и Ktor HTTP-клиент.
Серверная часть реализована на Spring Boot. Она обрабатывает поиск, фильтрацию, автодополнение, получение карточки публикации и обновление параметров ранжирования. Данные хранятся в MySQL.

## Формула ранжирования

**Модель ранжирования**
Итоговый рейтинг публикации рассчитывается на основе нескольких факторов:
- BM25 - текстовая релевантность;
- lambda - влияние возраста публикации;
- alpha - вес h-index авторов;
- beta - вес общего числа цитирований;
- gamma - вес влиятельных цитирований.
Такой подход учитывает не только совпадение с запросом, но и научную значимость, актуальность и авторитетность публикации.


Итоговая релевантность публикации рассчитывается как сумма текстовой релевантности BM25 и наукометрических факторов с учетом возраста публикации:

$$
Score(d, q) =
w_{BM25} \cdot BM25(d, q)
+
e^{-\lambda \cdot age(d)}
\cdot
(\alpha \cdot H_{norm}(d) + \beta \cdot C_{norm}(d) + \gamma \cdot I_{norm}(d))
$$

где BM25 рассчитывается по формуле:

$$
BM25(d, q) =
\sum_{t \in q}
IDF(t) \cdot
\frac{tf(t, d) \cdot (k + 1)}
{tf(t, d) + k \cdot (1 - b + b \cdot \frac{|d|}{avgdl})}
$$

$$
IDF(t) =
\ln \left(
\frac{N - n(t) + 0.5}{n(t) + 0.5} + 1
\right)
$$

Нормализованные наукометрические показатели:

$$
H_{norm}(d) = \frac{\ln(h(d) + 1)}{M_H}
$$

$$
C_{norm}(d) = \frac{\ln(c(d) + 1)}{M_C}
$$

$$
I_{norm}(d) = \frac{\ln(i(d) + 1)}{M_I}
$$

Обозначения:

- `d` - публикация;
- `q` - поисковый запрос;
- `Score(d, q)` - итоговый рейтинг публикации;
- `w_BM25` - вес текстовой релевантности;
- `tf(t, d)` - частота термина `t` в публикации;
- `IDF(t)` - обратная документная частота термина;
- `N` - общее число публикаций в выборке;
- `n(t)` - число публикаций, содержащих термин `t`;
- `|d|` - длина текста публикации;
- `avgdl` - средняя длина публикации в выборке;
- `k`, `b` - параметры BM25;
- `age(d)` - возраст публикации в годах;
- `lambda` - коэффициент временного затухания;
- `h(d)` - средний h-index авторов публикации;
- `c(d)` - число цитирований;
- `i(d)` - число влиятельных цитирований;
- `M_H`, `M_C`, `M_I` - максимальные логарифмические значения для нормализации;
- `alpha`, `beta`, `gamma` - веса h-index, цитирований и влиятельных цитирований.

**REST API**
| Endpoint | Метод | Назначение |
|---|---:|---|
| `/search` | GET | Поиск публикаций с фильтрацией и ранжированием |
| `/publication/{id}` | GET | Получение подробной информации о публикации |
| `/suggest` | GET | Автодополнение фильтров |
| `/params` | GET | Получение текущих параметров ранжирования |
| `/params/update` | POST | Обновление параметров ранжирования |

**Запуск**
Перед запуском сервера необходимо настроить подключение к MySQL:

`server/src/main/resources/application.yml`

> [!WARNING]
> База данных не входит в данный репозиторий

По умолчанию сервер ожидает базу данных **semantic_scholar_db**.

Клиент обращается к серверу по адресу, заданному в файле:
`composeApp/src/commonMain/kotlin/org/olgakhamzina/scientificlibrarythesis/network/createHttpClient.kt`.

Сейчас там указан host 10.0.2.2, адрес при необходимости нужно заменить на доступный адрес сервера.
